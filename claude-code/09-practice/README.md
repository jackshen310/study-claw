# 模块九：实战综合项目

> 本模块将前八个模块融会贯通，给出四类常见真实场景的“可执行流程”

---

## 9.1 从零到 PR：功能开发全流程

**目标**：为 Express API 增加“登录失败 5 次封禁 15 分钟”。

**流程**：
1. 准备 CLAUDE.md（模块六）
2. `--plan` 只读分析（模块四）
3. 产出方案 → 确认后执行（模块二 + 三）
4. 自动格式化/测试（模块八）
5. 生成 PR 描述

---

## 9.2 复杂 Bug 调试（跨模块）

**目标**：排查“支付成功但订单状态未更新”的竞态问题。

**关键策略**：
- 搜索优先：Glob → Grep → Read（模块三 + 五）
- 并行探索：Subagent 分工排查（模块七）
- 修复后必验证（Bash 测试）

---

## 9.3 大规模重构

**目标**：拆分 1.5 万行 monolith，保持测试通过。

**关键策略**：
- Subagent 并行分析依赖（模块七）
- Worktree 隔离并行修改
- CLAUDE.md 约束重构规则（模块六）

---

## 9.4 高质量 Prompt 模板

```
# 背景
[当前状态]

# 目标
[要实现的结果]

# 约束
- [不能改变的点]
- [必须遵守的规范]

# 验收标准
- [ ] 条件 1
- [ ] 条件 2

# 参考
[相关文件/链接]
```

---

## 9.5 专家级工作清单

**开始前**
- [ ] 更新 CLAUDE.md
- [ ] 配置权限规则
- [ ] 必要时先用 `--plan`

**执行中**
- [ ] 任务拆分 + 并行
- [ ] 避免一次性大文件读取
- [ ] Context 接近上限时 `/compact`

**结束后**
- [ ] 复查 Hook 自动动作
- [ ] 确认测试/构建结果
- [ ] 记录关键经验到 CLAUDE.md

---

## 9.6 核心要点总结

- 实战的关键是“流程化”：准备 → 分析 → 执行 → 验证 → 交付。
- 高质量 Prompt = 背景 + 目标 + 约束 + 验收标准。
- Subagent + Hooks 是规模化协作的加速器。
